# Installation Guide

**Suites is a versatile and flexible meta-framework, compatible with a wide range of testing and dependency injection
(DI) frameworks. This guide provides step-by-step instructions for installing Suites and highlights important
compatibility and configuration details.**

## Quick Start

To install Suites, select the adapter and testbed that align with your project's DI framework and preferred testing
library. For instance, if you pick Jest as your testbed, you can:

<Tabs groupId="suites-adapter" className="adapters-tabs" queryString="suites-adapter">
  <TabItem attributes={{className: 'nestjs'}} value="nestjs" label="NestJS" default>
    <CodeBlock language="bash">
      {`$ npm install --save-dev @suites/adapters.nestjs @suites/jest`}
    </CodeBlock>
  </TabItem>
  <TabItem attributes={{className: 'inversifyjs'}} value="inverisfyjs" label="InversifyJS">

    </CodeBlock>
  </TabItem>
</Tabs>

## Compatibility

Suites supports a variety of combinations of DI frameworks and testing libraries. Depending on your project's specific
needs, you can mix and match "adapter" and "testbed" packages from Suites.

### Monorepos

Suites is fully compatible with Monorepos, accommodating projects that utilize different DI frameworks. Ensure you
install the correct adapter for each workspace within your Monorepo. In some cases, multiple adapters may be necessary
to support different frameworks within your project.

### Instructions for Vitest

When integrating Suites with Vitest, additional configuration is required. Vitest typically uses `esbuild` for
TypeScript interpretation, which doesn't support `emitDecoratorMetadata` - a feature extensively used by Suites for
reflecting class dependencies. To overcome this, switch to using `@swc/core` via a plugin, which
supports `emitDecoratorMetadata`. For detailed guidance, see the [Vitest documentation](https://vitest.dev/docs/).
